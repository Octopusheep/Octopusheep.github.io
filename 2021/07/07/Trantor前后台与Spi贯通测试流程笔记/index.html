Trantor前后台与Spi贯通测试流程笔记

预置开发环境（win/mac）

  • Git
  • JetBrain Intellij IDEA CE 20.2-21.1版本
  • JetBrain Pycharm
  • JDK 1.8
  • IDEA SG插件
  • Postman

上述资源需要正确配置，可以根据网上相关教程快速补齐

IDEA过往版本可以在IDEA历史版本下载页中找到

预置资源文件

  1. 无法通过git命令clone拙远哥的qa_auto_patch的py脚本，需要在群文件中下载

  2. maven仓库的xml配置文件需要在群文件中下载，文件名为settings_quanxi.xml

  3. IDE的sg插件需要在群文件中下载，应对不同的IDE版本，需要安装对应的sg插件版本

  4. sg插件的环境需要配置如下信息，以全息1为例

  1. postman接口测试json副本需要找拙远哥或其他有副本的同学获取

通用问题tips

  1. qa_auto_patch需要解析yaml格式的配置文件，报错提示为ModuleNotFoundError: No module named 'yaml',需要在当前python脚本目录下通过pip安装yaml解析器

  1. 执行主函数若遇到FileNotFoundError: [Errno 2] No such file or directory: 'output\\front.diff'错误，在工程路径下新建一个output文件夹即可

  2. 作为省侧登陆时，创建中台应用时遇到非平台方，不能创建中台应用的提示（当前只能选择普通应用和通过应用库安装中台应用）

  3. SG插件登陆时若遇到如下错误, 那就是环境配置的cas地址填错了，每次修改环境配置后必须点击修改方可生效

windows上会遇到的问题

  1. qa_auto_patch主函数运行时会GBK解码失败的错误，通用解决方案是运行脚本时添加-X utf8参数指定使用utf-8编码

  2. 如果上述依旧报错，将patch_modifier.py文件中的open()函数指定encoding参数

mac上会遇到的问题

  1. 21.2版本的IDE无法兼容sg插件，需要安装20.2版本的IDE（该issue发现时间为2021/6/17，截至当前可能已经解决，则忽略）

测试流程侧思考

  1. 创建应该时填入的group id为java的包命名格式com.alitest.stddemo4。如果应用为stddemo8，应用的group id应修改为com.alitest.stddemo8

  2. 中台应用的功能域有四个模块，包括前台模块、子域模块、业务能力模块、SDK模块；普通应用没有业务能力模块、SDK模块

  3. 子域模块-模型-实体就类似于建立一个类似持久化数据库表一样的DO对象，该DO对象具有很多字段，类似数据库的表，可以在前端页面中方便地对表中的各字段做定义和编辑

  4. 实体定义完成后，可以通过当前模型版本中的版本详情，将对实体对象的修改，发布到maven仓库中

  5. 当前模型版本发布到maven仓库后，即可新增发布计划（子域模块-模型侧），并为该发布计划设置分库的数据key，一般trantor-run_default选项

  6. 是否可以这么理解，VO和DO实际上只是应对前端和后端模块会使用到的数据模型的不同叫法，在前端模块中数据模型具体实现和后端肯定有所不同，但是通过MSC的能力可以快速在前后端创建可以匹配的数据模型，例如在后端快速建表建字段，前端快速生成访问接口。叫法不同是一方面，另一方面他们的定位和在架构中的分层也不同。我们的demo比较简单，所以vo和do刚好字段一样。真实情况下大概率是不一样的，vo代表了页面要展示的字段，do代表领域模型的实体，一般情况下vo的数据可能来自多个do
