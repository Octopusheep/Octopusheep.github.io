what is Selenium

引用官方的话

Selenium is an umbrella project for a range of tools and libraries that enable and support the automation of web browsers.It provides extensions to emulate user interaction with browsers, a distribution server for scaling browser allocation, and the infrastructure for implementations of the W3C WebDriver specification that lets you write interchangeable code for all major web browsers.

基本就是说：

  1. Selenium是一个包含了各种工具和库的项目，用来实现浏览器的自动化

  2. 它提供一些插件，用以实现在浏览器中模拟用户操作

what is WebDriver

WebDriver drives a browser natively, as a user would, either locally or on a remote machine using the Selenium server

就是说WebDriver可以像真实用户一样，使用Selenium服务来在本地或远程驱动浏览器。可以说WebDriver是Selenium的核心能力

了解Selenium的工作流

在官方文档中有这么一段话，我觉得很重要

WebDriver has one job and one job only: communicate with the browser via any of the methods above. WebDriver does not know a thing about testing: it does not know how to compare things, assert pass or fail, and it certainly does not know a thing about reporting or Given/When/Then grammar.This is where various frameworks come in to play. At a minimum you will need a test framework that matches the language bindings, e.g. NUnit for .NET, JUnit for Java, RSpec for Ruby, etc.

大概就是说WebDriver只做了一件事就是通过你定义的方法去和浏览器通信。因此WebDriver是没有能力知道任何与测试相关的细节，例如常见的参数比较、断言等。WebDriver只负责通信，因此实际上Selenium的工作流包含了一下几个模块

  1. TestFramework模块

    这个模块包含了WebDriver模块，即TestFramework中包含了各种与测试相关功能的库和函数，这些函数通过WebDriver模块与浏览器通信

  2. WebDriver模块

    TestFramework中的函数传递到该模块后，该模块会将信息传递给下一个模块，即Driver模块

  3. Driver模块

    即和浏览器直接对接的Driver，不同的浏览器需要下载使用不同的Driver，上述步骤中已有提及

  4. Browser模块

    即浏览器本身，测试操作流最终会到达浏览器进行操作,相应操作结果会沿相同路径返回给TestFramework模块

至此Selenium的工作流完成闭环

官方文档中提到：

However, if there are no buttons outside of the iframe, you might instead get a no such element error. This happens because Selenium is only aware of the elements in the top level document. To interact with the button, we will need to first switch to the frame, in a similar way to how we switch windows. WebDriver offers three ways of switching to a frame.

即如果DOM元素实在iframe框架中，你是没办法直接通过driver.find_element(By.TAG_NAME, 'button')之类的函数直接找到该DOM元素的，因为 Selenium 只便利顶层文档中的元素。要与iframe中的元素交互，我们首先需要切换到框架

  • WebDriver 提供了三种切换到框架的方式：

    1. Using a WebElement

    2. Using a name or ID

    3. Using an index

    • Leaving a frame

定位元素相关方法

locating elements in WebDriver is done on the WebDriver instance object. The findElement(By) method returns another fundamental object type, the WebElement.

官方文档中提到WebDriver对象的findElement(By)方法返回的是一个WebElement对象。概念上也很好理解:

  • WebDriver代表的是浏览器
  • WebElement代表的是一个特定的DOM节点

元素定位策略

WebDriver中具有八种内置的元素定位的策略：

  1. class name

    Locates elements whose class name contains the search value (compound class names are not permitted)

    根据类名搜索

  2. css selector

    Locates elements matching a CSS selector

    匹配css选择器搜索

  3. id

    Locates elements whose ID attribute matches the search value

    根据id属性搜索

  4. name

    Locates elements whose NAME attribute matches the search value

    根据name属性搜索

  5. link text

    Locates anchor elements whose visible text matches the search value

    根据link属性搜索

  6. partial link text

    Locates anchor elements whose visible text contains the search value. If multiple elements are matching, only the first one will be selected.

    根据link属性搜索，匹配第一个符合条件的玄色

  7. tag name

    Locates elements whose tag name matches the search value

    根据tag属性搜索

  8. xpath

    Locates elements matching an XPath expression

    匹配XPath表达式搜索

元素定位策略选择上的Tips

  1. 当HTML的元素ID唯一且可以稳定预测的情况下，优先使用ID匹配策略，因为该策略不需要遍历复杂的DOM树

  2. ID不唯一时，优先使用css selector匹配策略，Xpath匹配策略虽然和css selector匹配策略功能相似，但是语法复杂，很难debug

  3. linkText和partialLinkText匹配策略仅适用于链接元素，此外它们在WebDriver内部调用XPath选择器

  4. tag匹配策略不建议使用，因为页面上经常会出现多个相同标签的元素。该策略常用于调用返回元素集合的findElements(By)方法

  5. 定位符尽可能短小并可读因为遍历DOM是一项代价高昂的操作，缩小搜索范围越多越好

注意到的区别

我注意到Selenium在官方文档中获取WebDriver对象时有两种方式

  1. 直接新建driver实例赋值

  2. 使用with关键字创建实例

Python’s WebDriver now supports the python context manager, which when using the with keyword can automatically quit the driver at the end of execution.

官方文档中提到python的webdriver支持python上下文管理器，可以使用with…as…的语法创建实例并赋值，这样可以在代码块执行完毕后自动执行driver.quit()
