解决windows环境下Postman转译curl命令无法执行的issue

问题背景

了解到部分客户环境存在物理机性能较弱或内网环境无法连接外网等问题，导致无法使用postman，需要将postman请求转义为cURL命令，放在终端中执行

由于windows环境存在对于postman导出格式不兼容的问题，特此文档说明

解决方案

  1. 进入postman,使用code snippet将对应请求转义为cURL请求

    需要注意如下几个参数：

    • Generate multiline snippet [Y]

      该参数将cURL命令分割为多行

    • Use long form options [Y]

      该参数开启时使用全称参数代替简写，开不开启都可以

    • Line continuation character [^]

      该参数选择换行连续符号，windows环境选择^

    • Quote Type [“]

      windows环境下，引用参数部分须使用双引号，参数内双引号需要使用/转义符

AirTest学习笔记

what is AirTest

李元哥分享的教程里提到：

AirTest自动化测试框架基于Selenium和python语言，通过调用Chrome浏览器Devtools Protocol协议，将用户操作解析成python脚本。

我理解主要是通过AirTest库，可以提供以下能力：

  1. 通过Chrome浏览器的Devtools协议，录制用户操作

    通过将用户操作转换成符合Airtest库调用链的方法，完成操作记录

  2. AirTest库中的调用链方法获取Selenium的WebDriver实例，操纵浏览器

    将第一部中的录制脚本通过Sellenium-Airtest转换为Selenium的WebDriver实例可以执行的方法，即可完成UI自动化步骤

  • 总体来说AirTest在脚本录制方面比较方便，另外在自动化测试结束后生成的报告有多种模版可以选择。

    但最后AirTest依旧是依赖Selenium的核心组件WebDriver去控制浏览器操纵DOM节点实现

How Does the Internet work?

Before Reading

该篇文章由Rus Shuler于1998年完成，并在2002年更新文章内容，浅析了互联网（Internet）的工作原理，我认为是一篇极其优秀的科普文章，从tcp/ip协议族的抽象层面和硬件层面（Protocol Stack）以及基础设施层面（Infrastructure）梳理了数据在互联网上的流动，并生动的给出了图示用于讲解互联网的硬件设施以及常用应用层协议数据传输过程。

这篇文章贼鸡儿棒，比大学时计算机网络的老师讲的强多了，这也是我翻译这篇文章的初衷。比起枯燥敷衍的说一遍OSI七层网络模型，听的人云里雾里,不如像这篇文章里分四层，从每一层的作用入手，配合ping、routetrace等工具详细讲解，这样网络的概念理解起来才会容易。能否把知识以别人能够听懂、能够理解的方式传授出去，我觉得是我们这个时代教育里极其欠缺的。

RESTful风格API初探

设计RESTful风格的API需要遵守以下三个原则

  1. 资源用URL来定位
  2. 不同类型的操作用不同的Method
  3. 返回码必须严格遵守HTTP协议规定的返回码

请求处理逻辑

当端口捕获请求后，按以下顺序执行：

handler -> validation{1.request 2.user} -> business logic -> response

  1. 端口捕获请求交给handlers，根据请求不同进入不同的handler
  2. handler中处理函数先进行validation，包括
    • 请求是否合法
    • 当前用户是否合法
  3. 业务逻辑执行
  4. 返回结果

Go语言常用工具链浅析

Go语言的build-in工具链异常强大，我简单总结一下test和benchmark的用法

go test

test用于测试代码，需要遵从以下几点原则

  1. 测试文件命名XXX_test.go

  2. 测试文件需要import testing

  3. 测试函数命名TestXXX

  4. 测试函数入参t *testing.T

  5. 函数打印错误信息t.Errorf(“message”)

  6. 测试函数如需跳过，可使用t.SkipNow()且必须放在测试函数第一行

  7. t.Run来执行subtest控制test输出和test顺序
